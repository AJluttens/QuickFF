1. qff.py – Main script to run QuickFF calculation¶

Usage: qff.py [options] fn1 [fn2 [...]]

This script will apply QuickFF to derive a covalent force field for the given
system. The arguments fn1, fn2, ... represent all input files that specify the
system and the ab initio reference data. Files later in the list overwrite
earlier files

Options:
  -h, --help            show this help message and exit
  --scoop               Flag to enable parallelisation using SCOOP. With
                        SCOOP, the command to run QuickFF is slightly
                        different, the absolute path to quickff.py should be
                        used. For example, to run on 4 cores: python -m scoop
                        -n4 /path/to/%prog --scoop [options] fns
                        [default=False]
  -s, --silent          Swith of all logging completely, overwrites all other
                        verbosity options. [default=False]
  -v, --verbose         Increases verbosity, is overwriten if SILENT is
                        switched on. [default=False]
  -l LOGFILE, --logfile=LOGFILE
                        Redirect logger output to a file with the given name.
                        [default=none]

  General:
    General specifications of the program

    -m PROGRAM_MODE, --program-mode=PROGRAM_MODE
                        Specify the program mode which defines the set of
                        instructions that will be executed. Allowed strings
                        are the names of the program classes defined in
                        quickff/program.py, which are: MakeTrajectories,
                        PlotTrajectories, DeriveDiagFF, DeriveNonDiagFF (case
                        sensitive). [default=DeriveNonDiagFF]
    --fn-traj=FN_TRAJ   Name for the file to read/write the perturbation
                        trajectories from/to. If the given file exists, the
                        trajectories are read from the file. Otherwise, the
                        trajectories are written to the given file.
                        [default=none]
    --only-traj=ONLY_TRAJ
                        Construct the perturbation trajectory only for the
                        terms with the given basenames. This options is only
                        applied in the MakeTrajectories program.
    -e, --ener-traj     Plot the various energy contributions along the
                        perturbation trajectories to. [default=False]
    -x, --xyz-traj      Write the perturbation trajectories in XYZ format.
                        [default=False]
    --suffix=SUFFIX     Suffix that will be added to all output files.
                        [default='']

  Force Field:
    All options related to the definition and derivation of the force
    field.

    --ei=EI             A Yaff parameters file defining the electrostatic
                        contribution of the force field. [default=none]
    --vdw=VDW           A Yaff parameters file defining the van der Waals
                        contribution of the force field. [default=none]
    --covres=COVRES     A Yaff parameters file defining a residual
                        contribution to the covalent part of the force field.
                        [default=none]

  System:
    All options related to the definition of the system.

    --ffatypes=FFATYPES
                        Assign atom types in the system, through either on of
                        three possible routines. If FFATYPES is a string equal
                        to either low,medium, high or highest, atom types will
                        be assigned through the automatic built-in detection
                        (see documentation). If FFATYPES is a different
                        string, it is assumed to be the name of a file
                        containing a list of atype names and Yaff ATSELECT
                        rules to defined the atom types. If FFATYPES is None,
                        the atom types are assumed to be defined in the input
                        files. [default=none]

2. qff-input-ei.py – generate Yaff parameter file for electrostatics¶

Usage: qff-input-ei.py [options] fn_sys fn_hdf5 path

This script will read the atomic charges from the input arguments and make the
Yaff parameters file for use with the QuickFF option --ei. fn_sys is any file
from which the system can be extracted (MolMod CHK, Gaussian FCHK, XYZ, ...).
Path represents the path to the charges, which are assumed to be stored in the
file fn_hdf5 in the dataset /path/charges.

Options:
  -h, --help           show this help message and exit
  --ffatypes=FFATYPES  Assign atom types in the system, through either on of
                       three possible routines. If FFATYPES is a string equal
                       to either low,medium, high or highest, atom types will
                       be assigned through the automatic built-in detection
                       (see documentation). If FFATYPES is a different string,
                       it is assumed to be the name of a file containing a
                       list of atype names and Yaff ATSELECT rules to defined
                       the atom types. If FFATYPES is None, the atom types are
                       assumed to be defined in the input files.
                       [default=none]
  --gaussian           Use gaussian smeared charges. The radii are taken from
                       the dataset /path/radii if it exists, otherwise the
                       radii are estimated according to the procedure of Chen
                       et al. See ``quickff.tools.get_ei_radii`` for more
                       info.
