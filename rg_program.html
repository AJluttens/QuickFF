3. Module quickff.program – Management of the entire program¶

class quickff.program.Program(system, model, fn_traj=None)[source]¶

The central class to manage the entire program.

Arguments

system
An instance of the System class and contains all the system information
model
An instance of the Model class and contains all the info to define the total PES and its electrostatic contribution.

Optional Arguments

fn_traj
A file name to store the perturbation trajectories to or to read the trajectories from if the file exists. The trajectories are stored/read after Pickling.
estimate_from_pt(trajectories, skip_dihedrals=True, verbose=True)[source]¶

Second Step of force field development: calculate harmonic force field parameters for every internal coordinate separately from perturbation trajectories.

Arguments

trajectories
A dictionairy containing numpy arrays representing perturbation trajectories for each icname.

Optional Arguments

skip_dihedrals
If set to True, the dihedral ff parameters will not be calculated.
generate_trajectories(skip_dihedrals=True, verbose=True)[source]¶

Generate a perturbation trajectory for all ics (dihedrals can be excluded and store the coordinates in a dictionary.

Optional Arguments

skip_dihedrals
If set to True, the dihedral ff parameters will not be calculated.
plot_pt(icname, start=None, end=None, steps=51, verbose=True)[source]¶

Generate the perturbation trajectories and plot the energy contributions along the trajectory for all ics with a name compatible with icname.

Arguments

icname
A string describing for which ics the perturbation trajectories should be generated.
refine_cost(verbose=True)[source]¶

Second step of force field development: refine the force constants using a Hessian least squares cost function.

run()[source]¶

Run all steps of the QuickFF methodology to derive a covalent force field. This method returns an instance of the class quickff.fftable.FFTable, which contains all force field parameters.
